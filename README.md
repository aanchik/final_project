# Применение технологий Retrieval-Augmented Generation для повышения эффективности работы персонала службы авиационной безопасности

## Описание проекта

Проект представляет собой специализированную RAG-систему (Retrieval-Augmented Generation), предназначенную для автоматизации поиска информации в нормативно-правовых актах, регламентах и инструкциях САБ. Система позволяет сотрудникам получать точные ответы на рабочие вопросы, исключая субъективные трактовки и опираясь строго на предоставленную базу знаний.

---

# Архитектура системы

Система построена на принципе гибридного извлечения данных, что обеспечивает баланс между семантическим пониманием запроса и точным поиском ключевых терминов.

## Структура проекта

```text
ANNY/
├── chroma_db/          # Векторная база данных (ChromaDB)
├── docs/               # Нормативная документация (.txt)
├── auto_check.py       # Система автотестирования точности (Accuracy)
├── auto_report.txt     # Отчеты по результатам тестирования
├── config.py           # Конфигурация и глобальные настройки
├── hybrid_retriever.py # Гибридный алгоритм поиска (FAISS + BM25)
├── index.html          # Интерфейс пользователя
├── indexer.py          # Скрипт индексации документов
└── rag_simple.py       # Ядро системы (поиск + взаимодействие с LLM)
```

---

# Описание компонентов

## `config.py`

Центральный файл настроек. Здесь хранятся пути к данным, параметры нарезки документов (`chunk size` / `overlap`) и настройки моделей.

## `indexer.py`

Модуль подготовки базы знаний. Выполняет парсинг текстовых файлов, разбивает их на смысловые фрагменты с использованием `RecursiveCharacterTextSplitter` и создает векторный индекс для эффективного поиска.

## `hybrid_retriever.py`

Реализация гибридного поиска. Сочетает методы FAISS (семантический поиск по векторам) и BM25 (статистический поиск по ключевым словам) для достижения максимальной релевантности выдачи.

## `rag_simple.py`

Логическое ядро системы. Отвечает за извлечение контекста из базы и формирование промптов для локальной LLM (Ollama), обеспечивая экспертный стиль ответов со ссылками на источники.

## `auto_check.py`

Инструмент контроля качества. Автоматически прогоняет систему через набор заранее подготовленных кейсов, вычисляет точность (`Accuracy`) и выявляет области, требующие доработки.

---

# Принцип работы

## 1. Индексация

На этапе подготовки документы из папки `docs` проходят процесс токенизации и векторизации. Каждый фрагмент снабжается метаданными об источнике.

## 2. Обработка запроса

При поступлении вопроса система выполняет векторный поиск по базе. Найденные релевантные фрагменты («контекст») передаются в LLM.

## 3. Генерация ответа

Модель анализирует контекст и формулирует ответ в соответствии с ролью «Эксперт САБ», строго следуя предоставленным инструкциям и исключая галлюцинации.

# Быстрый старт

## Подготовка

Убедитесь, что все актуальные приказы и регламенты загружены в папку `docs` в формате `.txt`.

## Индексация данных

Запустите скрипт для обновления векторной базы:

```bash
python indexer.py
```

## Верификация

Для оценки текущей эффективности системы запустите автоматический тест:

```bash
python auto_check.py
```

Результаты будут доступны в файле `auto_report.txt`.

# Как запустить систему с телефона и других устройств

## Почему `localhost` не работает на телефоне

Адрес:

```text
http://localhost:8000
```

работает только на компьютере, где запущен сервер.

Для подключения с телефона необходимо использовать локальный IP-адрес компьютера в одной Wi-Fi сети.

---

# Пошаговый запуск системы

## 1. Установить Python

Скачать и установить Python:

* Windows: https://www.python.org/downloads/
* Во время установки обязательно включить:

```text
Add Python to PATH
```

Проверка установки:

```bash
python --version
```

---

# 2. Установить Ollama

Скачать Ollama:

* https://ollama.com/download

После установки запустить модель:

```bash
ollama run llama3
```

или другую используемую модель.

---

# 3. Установить зависимости проекта

Перейти в папку проекта:

```bash
cd ANNY
```

Установить библиотеки:

```bash
pip install -r requirements.txt
```

---

# 4. Загрузить документы

Поместить все `.txt` документы в папку:

```text
docs/
```

---

# 5. Создать индекс базы знаний

Запустить индексацию:

```bash
python indexer.py
```

После завершения появится база:

```text
chroma_db/
```

---

# 6. Запустить сервер

Запуск:

```bash
python app.py
```

После запуска в консоли появится примерно такое сообщение:

```text
Uvicorn running on http://0.0.0.0:8000
```

---

# 7. Узнать IP-адрес компьютера

## Windows

Открыть CMD и выполнить:

```bash
ipconfig
```

Найти строку:

```text
IPv4 Address
```

Например:

```text
192.168.0.15
```

---

## Linux / macOS

В терминале:

```bash
ifconfig
```

или:

```bash
ip a
```

---

# 8. Подключение с телефона

Телефон и компьютер должны быть подключены к одной Wi-Fi сети.

На телефоне открыть браузер и перейти по адресу:

```text
http://192.168.0.15:8000
```

где:

```text
192.168.0.15
```

— IP-адрес компьютера.

После этого откроется веб-интерфейс RAG-системы.

# Если сайт не открывается

## Разрешить доступ через брандмауэр Windows

При первом запуске Python может появиться окно:

```text
Разрешить доступ к сети?
```

Нужно нажать:

```text
Разрешить
```

---

## Проверить порт 8000

Иногда порт блокируется антивирусом или firewall.

Можно изменить порт в `app.py`:

```python
uvicorn.run(app, host="0.0.0.0", port=8000)
```

например:

```python
uvicorn.run(app, host="0.0.0.0", port=8080)
```
